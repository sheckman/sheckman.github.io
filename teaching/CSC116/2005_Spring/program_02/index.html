Software Overview

In this assignment you will be designing a program to output a dollar amount in the range of 0.00 to 999999.99 as English text as it would appear on a check.

The program must match the output of the included applet exactly, so please take note of the applet output and test your code against it.

You will be only designing the model code. The GUI code is provided. Please click here to download the GUI once you have COMPLETED your model code. The GUI will not compile or run unless your model methods match the method signatures outlined below exactly.

Requirements

Your program must create the class CheckWriterModel and implement the following public methods:

public CheckWriterModel()
Null constructor, which must initialize all of your instance variables. This must set the initial value of the check amount to 0.00
public double getCheckAmount()
Method, which will return the value of check amount as a double when called.
public void setCheckAmount(double)
Method, which will accept a single double as a parameter and set the value of the check amount to that double. This method should NOT change the value of the check amount if it is provided a negative value. You should truncate any fractional pennies.
public String getCheckWords()
Method, which will return the value of check amount as English text as outlined below.
public static void main(String[] args)
Main method which should contain your test code to show you have tested your model.

You may not have ANY OTHER PUBLIC ITEMS in your code. All other items must be private. Your code must run with the given GUI and must meet the specifications below.

Specification

Your program must accept a double value in the range of 0.00 to 999999.99 and translate it to English text.

The setCheckAmount method will be how we provide your object with a value. This method must handle two special cases:

  • Negative values
  • Values with fractional pennies
If a negative value is provided the method should not change the instance variable. If a value with a fractional penny amount, such as 5.955, is provided then you should truncate the fractional penny. In this example that means a setCheckAmount(5.955) would result in getCheckAmount() returning 5.95.

The getCheckAmount method must return the value of the check amount that was set with the setCheckAmount method.

The constructor will initially set the check amount to 0.00. Your constructor is where all of your private instance variables must be initialized.

The getCheckWords method is the method which will translate the double value in to text. It must return a String containing proper English text. For the value 123456.78 the program would return the String:
One Hundred Twenty Three Thousand Four Hundred Fifty Six Dollars and 78 Cents

Your program must handle the following special situations:

  • The word "Dollars" must be plural except when dealing with a single dollar. In that case it must be "Dollar".
  • The word "Cents" must be plural except when dealing with a single cent. In that case it must be "Cent".
  • All cent outputs must be 2 digits
This means 1.01 would read "One Dollar and 01 Cent". The values between 1.00 and 1.99 will read with "One Dollar".

Your program must also take in to account having values less than one dollar. In this case you must print "Zero Dollars" and then the cents portion of the output. This means 0.75 would print as "Zero Dollars and 75 Cents".

The first letter of each word except for the "and" must be in a capital letter. Your output must match the provided applet EXACTLY so please test against the applet.

The provided GUI will only function once your model code meets the method signatures. If you attempt to compile the GUI and it does not compile that means your model code is not setup according to specifications.
