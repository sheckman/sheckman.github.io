Program 1 Grading Guidelines: Pizza Parlour

Program Functionality

The first test of functionality is to run the grader. If the code passes the grader then you can assume all points are to be awarded. If the code fails one or more tests for a method you should check the following.

First make sure the student is getting the input values from the parameters. Some students will mistakenly use the wrong inputs. If the math is correct but the wrong input is used only deduct the 4 points for use of the parameters.

Second make sure the student is performing the math correctly. Regardless of the output format or the correct use of parameters the student could be doing the correct math. If the math is wrong deduct all 10 points.

Finally check the output wording and format. If the output contains the wrong words or punctuation deduct 2 points. Again it is possible for the correct math and parameter use to result in the wrong output.

If you have any questions regarding grading contact one of the lecture TAs or administrators for assistance. Generally a single error in a method should result in one deduction.

Testing

The student should have included two additional test cases in the main method which each test every method. The student should receive 1/2 point for each method being tested a single time.

The test cases should receive credit only if they are unique inputs for the method. If the student uses the same inputs for all tests of a method then no credit should be given for duplicate cases.

JavaDoc

JavaDoc in the program should meet the course style guidelines. Each subsection should receive all or none of the points on the gradesheet. You should clearly document on the student's hard copy the correct way to perform JavaDoc if any points are deducted.

For each method the student must have included documentation for the method and have documented each parameter in the JavaDoc. If the JavaDoc is missing the method description or has the wrong format then grade should be 0/1 for the method description.

If the @param tags are missing or incorrect in format 0/1 should be awarded for the parameter description for the method.

The class description requires that the student have correctly formatted JavaDoc for the class to receive the 3 points. The @version tag is worth 1 point and the @author tag is worth 1 point.

Limitations

If the program fails to compile without modification the code can ONLY earn points for having functional test cases and JavaDoc. You still must grade those two section by hand.
